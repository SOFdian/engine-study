# 第二周学习计划：现代 C++ 基础重整

> 依据《52 周引擎岗学习路线》中的“第 2 周：现代 C++ 基础重整”进行细化。本周的目标是从“会写 C++”转向“理解 C++ 内存模型与高性能资源管理”，为后续引擎模块开发夯实底层功力。

------

## 一、 本周目标

本周的核心任务是**消除对 C++ 内存管理的恐惧与误解**，建立“所有权”和“零开销抽象”的概念。

围绕原路线要求，本周需要完成以下 4 个结果：

1. **资源封装类**：实现基于 RAII 的 `FileHandle` 和 `MemoryBuffer`。
2. **性能计时器**：实现一个基于 RAII 的 `ScopeTimer`，用于后续所有模块的性能检测。
3. **实验报告**：通过 `Trace` 类观察并记录拷贝构造与移动构造在不同场景下的触发频率。
4. **面试答辩准备**：能准确解释值类别（Value Categories）与智能指针的边界。

------

## 二、 本周学习原则

### 1. “内存视角”优先

每写一行代码，都要反问：这个对象在内存的哪个区？谁负责释放它？这里有没有产生不必要的内存拷贝？

### 2. 用 Godbolt 说话

对于不确定的语法特性（如 `std::move` 是否真的移动了数据），使用 [Compiler Explorer (Godbolt)](https://godbolt.org/) 查看汇编输出，验证编译器是否做了优化。

### 3. 拒绝过度设计

在引擎开发中，`std::shared_ptr` 不是万灵药。本周要学会区分“所有权（Ownership）”与“使用权（Access）”。

## 三、 第二周详细日计划

### Day 1：值类别与对象生命周期 (The Basics)

#### 今天学什么

- **值类别图谱**：什么是 `lvalue` (左值), `rvalue` (右值), `prvalue` (纯右值), `xvalue` (将亡值)。
- **生命周期**：局部对象、静态对象、堆对象的析构时机。
- **引用折叠** (Reference Collapsing) 基础。

#### 今天怎么学

1. 阅读 *Modern C++* 相关博客或《Effective Modern C++》Item 1-2。
2. 编写一个 `Trace` 类，在构造、拷贝、移动、析构中增加 `std::cout` 打印。
3. 实验：将 `Trace` 对象放入 `std::vector` 并触发扩容，观察打印结果。

#### 今天的产出物

- `Trace` 辅助类（用于后续一周的所有实验）。
- 一份关于“对象何时析构”的实验笔记。

------

### Day 2：RAII 与资源管理封装

#### 今天学什么

- **RAII 核心思想**：资源获取即初始化。
- **Rule of Five**：三/五/零法则。
- 显式删除（`= delete`）拷贝操作的意义。

#### 今天怎么学

1. 实现一个 `FileHandle` 类封装 `FILE*`。
2. 实现一个 `MemoryBuffer` 类，在构造时 `new` 内存，析构时 `delete`。
3. 练习：如何让一个类“只能移动，不能拷贝”。

#### 今天的产出物

- `FileHandle.hpp` (集成到 `engine_core/include`)。
- `MemoryBuffer.hpp`。

------

### Day 3：移动语义 (Move Semantics) 深度解析

#### 今天学什么

- **`std::move` 的本质**：它只是一个 `static_cast<T&&>`。
- **移动构造函数与移动赋值**：如何实现资源的“窃取”。
- **性能优势**：为什么移动 `std::vector` 是 $O(1)$ 而不是 $O(n)$。

#### 今天怎么学

1. 为 Day 2 的 `MemoryBuffer` 编写移动构造函数。
2. 使用 `std::swap` 实现移动赋值（Copy-and-Swap 变形）。
3. **验证**：在 Godbolt 中对比传值返回一个大对象时，开启和关闭移动语义的汇编差异。

#### 今天的产出物

- 支持移动语义的 `MemoryBuffer`。
- 笔记：移动语义如何避免引擎中的“大块内存搬运”。

------

### Day 4：万能引用与完美转发 (Perfect Forwarding)

#### 今天学什么

- **万能引用 (`T&&`)**：模板中的右值引用为何能同时接收左值和右值。
- **`std::forward`**：如何保持参数的原始属性。
- 模板实例化中的参数传递开销。

#### 今天怎么学

1. 编写一个简单的工厂函数模板 `MakeUnique<T>(Args&&... args)`。
2. 实验：如果不使用 `std::forward`，参数进入构造函数时会发生什么（属性退化）。
3. 阅读《Effective Modern C++》Item 23-25。

#### 今天的产出物

- 几个体现“完美转发”价值的示例代码（存入 `tests/`）。

------

### Day 5：现代智能指针的边界 (Ownership)

#### 今天学什么

- **`std::unique_ptr`**：独占所有权与零开销。
- **`std::shared_ptr`**：引用计数、控制块（Control Block）与原子操作开销。
- **`std::weak_ptr`**：解决循环引用（重要：引擎场景树的基础）。

#### 今天怎么学

1. 模拟场景树：父节点用 `shared_ptr` 指向子节点，子节点用 `weak_ptr` 指向父节点。
2. **性能实验**：对比 100 万次 `unique_ptr` 解引用与 `shared_ptr` 拷贝的耗时。
3. 思考：为什么引擎内部高频使用的指针往往是裸指针（Raw Pointer）？

#### 今天的产出物

- `tests/test_pointers.cpp`。
- 笔记：智能指针的选择矩阵。

------

### Day 6：实战：实现高精度 ScopeTimer

#### 今天学什么

- `std::chrono` 库的高级用法。
- 将 RAII 应用于性能统计。
- 格式化输出与静态变量的应用。

#### 今天怎么学

1. 在 `engine_core` 中实现 `Timer` 类。
2. 实现 `ScopeTimer`：构造时记录 `now()`，析构时记录 `now()` 并计算差值，通过 `LOG_INFO` 输出。
3. **应用**：用 `ScopeTimer` 测试 Day 3 中拷贝 vs 移动的实际运行时间差。

#### 今天的产出物

- `ScopeTimer.hpp` (这是后续 50 周非常高频使用的工具)。
- 一份真实的性能对比报告。
