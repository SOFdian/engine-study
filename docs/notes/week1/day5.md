这部分主要是Assert的代码实现

**Fail Fast（快速失败）**：Assert 的哲学是“与其带着错误的数值继续运行导致后面莫名其妙的崩溃，不如现在就停下来”。

**调试加速器**：当 `CORE_ASSERT` 触发时，如果你在 IDE（如 Visual Studio 或 CLion）中运行，它会直接弹窗并定位到出事的那一行代码。这比翻看数千行日志要快得多。

**性能开销**：由于 `CORE_ASSERT` 仅在 Debug 模式下生效，你可以在性能敏感的循环中大量使用它，而不用担心会拖慢正式发布版的运行速度。
